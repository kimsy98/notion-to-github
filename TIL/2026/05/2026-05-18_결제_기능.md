### ShedLock

- 분산 환경에서 스케줄러(Job/Scheduler) 작업이 여러 서버에서 동시에 실행되는 것을 막기 위한 라이브러리
- 1번만 실행되도록 보장
- 스토리지 기반 처리 : Lock 정보를 공용 스토리지(DB)에 저장 및 조회
- lockAtMostFor, lockAtLeastFor 파라미터로 Lock 유지 시간 제어
- shedLock 테이블 필요
```sql

    lock_until TIMESTAMP(3) NOT NULL,
    locked_at TIMESTAMP(3) NOT NULL,
    locked_by VARCHAR(255) NOT NULL,
    PRIMARY KEY (name)
```

3.2 구독 복구 시 추적 부족

시나리오: 사용자가 구독을 취소했다가 남은 기간 내에 다시 구독

// UserSubscriptionService.preparePaymentAndCheckEligibility()
if (subscription.isCanceledWithActiveRemainingPeriod(...)) {
if (subscription.getSubscriptionType() == subscriptionType) {
subscription.reactivateAsIs();  // ← 결제 없이 복구
} else {
subscription.reactivateWithTypeChange(subscriptionType);
}
return false;  // PortOne 결제 스킵
}

문제:

- PaymentAttempt: 생성됨 (PENDING) → 그런데 PortOne 호출 안 됨
- PaymentHistory: 생성 안 됨 ← 결제 없이 복구되었으므로
- 사용자에게 "구독이 복구되었습니다" 알림이 없음
복구 액션의 이력 추적이 필요한 이유:

1. 감사 (Audit): 언제, 누가, 어떤 조건으로 무료 복구되었는지
1. 고객 지원: "왜 결제 안 했는데 구독이 됐어요?" 문의 대응
1. 데이터 분석: 복구 패턴 분석 (취소 후 재구독 간격, 등급 변경 패턴)
