# 🚀 Claude Code 완전정복 통합 가이드 (입문~마스터)

> 💡 가이드 개요
Claude Code를 처음 시작하는 분들을 위한 초기 설정부터 실전 워크플로우, 고급 자동화(Hooks/Sub-Agents), 그리고 핵심 제어 파일인 CLAUDE.md 마스터 방법까지 총망라한 실무 밀착형 치트시트입니다.

## 1️⃣ Part 1. 입문: 초기 설정 & 필수 단축키

Claude Code의 기초적인 환경 설정과 작업 속도를 높여주는 단축키 및 명령어 모음입니다.

### ⚙️ 초기 설정

- 시작 위치: 반드시 프로젝트 루트 디렉토리에서 실행하여 Claude가 프로젝트 구조를 파악할 수 있게 하세요.
- /init 명령어: 최초 실행 시 프로젝트를 분석해 규칙과 구조가 담긴 claude.md 파일을 자동 생성합니다.
### ⌨️ 핵심 키보드 단축키

- Shift + Tab (모드 전환): Plan Mode(계획)와 Accept Mode(실행)를 전환합니다. 토큰 낭비와 엉뚱한 코드 생성을 막기 위해 새로운 작업은 항상 Plan Mode로 시작하세요.
- Escape (즉시 중단): Claude가 잘못된 가정을 하고 엉뚱한 코드를 짤 때 즉시 작업을 중지시킵니다.
- Escape 2번: 입력 내용 전체 삭제 또는 이전 입력 내용 복원.
- ! (터미널 명령): 프롬프트 앞에 !를 붙이면 대화를 끊지 않고 Bash 명령어(예: !npm run build)를 즉시 실행합니다.

## 2️⃣ Part 2. 실전: 생산성 극대화 워크플로우

현업에서 Claude Code를 사용할 때 효율을 극대화하는 실전 패턴입니다.

### 🧠 컨텍스트 & 토큰 관리

- 1 세션 = 1 피처 원칙: 한 세션에서 여러 기능을 연달아 구현하지 마세요. 작업이 끝나면 /clear로 초기화해야 응답 품질이 유지됩니다.
- 무거운 작업은 스크립트로: 대용량 로그 분석이나 데이터 마이그레이션은 대화창에서 시키지 말고 스크립트를 작성하게 한 뒤 결과만 요약해서 받으세요.
- Lazy Loading: CLAUDE.md에는 모든 내용을 다 적지 말고, @docs/api-spec.md처럼 참조 링크만 남겨 Claude가 필요할 때만 읽도록 유도하세요.
### 🔄 TDD 기반 스마트 코딩 패턴

작은 변경 → 테스트 → 린트 → 커밋 → 다음 단계로 넘어가는 짧은 루프를 유지하세요.

- 에러 처리: 에러 로그를 직접 해석하지 말고, 스택 트레이스 전체를 복사해서 그대로 던져주는 것이 훨씬 효과적입니다.
- 다른 AI 활용: 막히는 부분이 있다면 /export로 대화를 내보내서 ChatGPT나 Gemini에게 비평을 받아보세요.
### 🛠️ WAT 프레임워크 (복잡한 프로젝트 관리)

- Workflows (W): 코드를 짜기 전에 글로 작업 단계를 먼저 명확히 정의합니다 (Plan Mode 활용).
- Agents (A): Sub-agent를 띄워 린트, 테스트 등을 병렬로 처리하여 시간을 단축합니다.
- Tools (T): 거대한 스크립트 대신, 원자적(Atomicity)으로 쪼개진 작은 스크립트 여러 개를 조합하여 실행합니다.

## 3️⃣ Part 3. 심화: 고급 자동화 및 확장

Sub-Agent와 Hooks를 활용해 1인 개발자가 팀급 생산성을 내는 방법입니다.

### 🎯 Skills (SKILL.md)

- 반복되는 프롬프트(예: PPT 생성, 코드 리뷰 등)를 매뉴얼화하여 자동 트리거하거나 슬래시 커맨드로 호출하는 기능입니다.
- skill-creator 플러그인을 설치하면 모범 사례가 적용된 스킬을 쉽게 만들 수 있습니다.
### 🤖 Sub-Agent & 병렬 실행

- 메인 Claude 외에 별도의 작업 공간을 가진 도우미를 띄우는 기능입니다.
- 장점: 여러 작업을 병렬로 처리하여 시간을 단축하고 메인 컨텍스트 오염을 막습니다.
- /agents 명령어로 생성하며, 대량의 로그 탐색이나 독립적인 코드 리뷰를 맡길 때 유용합니다.
### ⚡ Hooks (자동화 엔진)

- 특정 이벤트(파일 저장, 도구 실행 등)에 맞춰 자동으로 지정된 액션을 실행합니다.
- settings.json에 규칙을 추가하며, 'PostToolUse' 이벤트에 린트나 포맷팅을 연결하는 식으로 활용 가능합니다.
### 💡 고급 팁

- 멀티 인스턴스 & Worktree: claude -w feature-branch 명령어로 Git 워크트리를 생성하여 패널별로 완전히 독립된 작업을 수행할 수 있습니다.
- 음성 입력: /voice를 통해 스페이스바를 누른 채 말하면 복잡한 요구사항을 1분 만에 전달할 수 있습니다.

## 4️⃣ Part 4. CLAUDE.md 마스터

CLAUDE.md는 단순한 설명서가 아니라 **"AI가 실수를 반복하지 않도록 막는 가드레일"**입니다.

### 🧱 4가지 위치 및 적용 범위

규칙의 범용성에 따라 적절한 위치에 분산 배치하세요.

1. 전역 (~/.claude/CLAUDE.md): 내 코딩 스타일 등 모든 프로젝트 공통.
1. 프로젝트 (<repo>/.claude/CLAUDE.md): 팀원 공유 규칙 (Git 커밋 필수).
1. 개인 로컬 (<repo>/CLAUDE.local.md): 개인 단축어나 임시 설정 (.gitignore 추가).
1. 폴더별 (<subdir>/CLAUDE.md): 특정 디렉토리에만 적용되는 특수 규칙.
### 📝 작성 제1원칙: "코드만 봐도 아는 건 쓰지 마라"

- "React를 사용합니다" (❌) -> "Hooks 밖에서 fetch 직접 호출 금지" (✅).
- 빌드, 테스트, 린트 명령어는 반드시 명시하세요. AI가 추측하느라 낭비하는 토큰을 극적으로 줄여줍니다.
### 🚀 High-ROI 워크플로우 (즉각 반영)

- Claude가 실수했을 때 즉시 **"이 실수 다시는 안 하게 CLAUDE.md에 반영해줘"**라고 지시하세요.
- 대화창에서 #을 입력하면 빠른 메모 추가가 가능합니다.
### 🧹 관리 및 다이어트

- claude-md-management 플러그인을 설치하여 월 1회 /revise-claude-md 명령어로 불필요한 규칙을 정리하고 다이어트하세요 (100줄 이하 권장).
- 규모가 커지면 @import ./docs/style.md 문법을 사용해 파일을 분할 관리하세요.
